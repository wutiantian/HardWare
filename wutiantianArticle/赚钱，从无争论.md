# 赚钱，没有争论

要赚钱，这个观点没有争论。

但无争论也是赚钱的前提之一。

以下是几大争论无休止的话题：

>1、硬件电路设计软件

用哪款电路图绘制软件？

这个话题要从上大学那会儿开始。必修课程中用 protel99SE ，全国电子竞赛用 Altium 15 。这两个软件其实还好，都是一家公司的。

到深圳工作，公司就是专门做这方面的外包项目。原理图用 cadence ，PCB layout 用 PADS 。干不干？不干就走。没办法，需要啥就是要会啥。等过了半年多，已经比较熟练得掌握业务，然而公司进入一个困难期，订单没有挑挑拣拣的选择，有那么个季度，一直接的都是甲方要求用 cadence 进行绘制的源文件要求。其实各个软件在输出时候都可以相互转化，但是客户要求绘制的时候就用指定的某软件就**必须按照客户要求来做**。

其实软件之间差异不是天差地别，只是顺手不顺手再加一些小区别而已，但是很影响生产效率，特别是专业的公司，对时效交付有要求的公司。

这三大软件阵营各有特色，适用于不同规模电路。初学者几乎体验不到效率问题，就像自行车和汽车在2米的启动范围内，差距不是很大。但专业的工作人员，接到的大规模电路是4K pin 以上，就很影响时间效率；当然初学者 4K pin 是没有能力完成的，更谈不上时效，说了他也不懂。

总之，甲方要什么，我们就给什么，没有商量的余地。

>2、编程语言与软件

用哪种语言和软件？

各位 EE专业的学生迫不及待发言：

最底层的用 51 单片机的青蛙发言了：当然是 C 语言，单片机支持 C 语言！

高一点儿平台的 stm32 开发板发言了：当然是 C++ 语言，我的开发板支持库开发便捷！

更高一点儿的 安卓方向发言了：当然是 JAVA 语言，我的安卓程序就是用 JAVA 的 API 调用，方便的很！

更新一点儿的 物联网方向 发言了：当然是 Python 语言，我用 micro python 语言开发快的很！

cortex A 的操作系统级 嵌入式 发言了：我用  C 语言 写内核，C++ 语言写 ，JAVA，Python 写算法，整套流程一个人就可以布置完成！

那来看看那些从业者过来人吧！

- 【汇编】华强北附近给老板看门店、做客服、还要为客户出方案的  “国产单片机三级分销”店员他的经历：入职前考察 C语言编程，入职后 五年从事 10元以下国产单片机批量芯片销售的他工资 5K 包住在深圳。每天为客户提供 “汇编51单片机”程序方案，他对于汇编是工作中新学的，在学校掌握了一些但不深入。没办法，这种三级分销商，要想卖出货就得提供好服务。用 C 语言写的话，客户拿着C程序转身去买另一款型号的单片机程序照抄，那不白忙活了？所以，就是要写大多数人看不懂不好抄的汇编，你不买这款芯片就别想用免费方案。十元以下的芯片能干什么工程？当然是小工程，路边小吃摊电炉子的简单应用场景等等。五年的从业经验让他收获什么？让他彻底放弃了汇编工作，转行了。第三年工作的他曾经就遭遇公司业绩差，经销商的老板注销了公司，去其他地方开店了，他不得不重新找工作。由于他一直都做小项目，没有大工程也不会操作系统级的贵芯片，所以还是找了另一家汇编做同样的事，当然，结局还是一样，干两年就被辞退了。他的第一年、第三年、第五年水平是一样的，技术没有太大的提升，尽管他一直都会买些书来学，但是好汇编书毕竟不多，而且他的编程应用场景受限，做不了大的，没有核心竞争力，涉及面也窄。

## 各种语言的发展

在第一句，我想提一下。即使是一种计算机语言，对应不同的编译器厂商软件，也有不同编译方法和标准。例如： C 语言随着时代发展有：C99标准、ANSI C标准等，而编译器开发商则是在标准之上又进行了扩充，以满足用户的额外需求。倘若连编译器都不知道，又怎么会知道正确的结果是什么，连结果目标都不知道还编程个啥？
